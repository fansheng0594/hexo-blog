PHP 超文本预处理器

PHP的三个主要领域：

  • 服务端脚本。需要PHP解析器、web服务器、web浏览器
  • 命令行脚本。仅需要PHP解析器来执行
  • 桌面应用程序。PHP-GTK

PHP支持绝大多数的web服务器，数据库，系统

超全局变量 $_SERVER

Post 并不会比 get 更安全。两者之间的选择原因有多种，一是 url 往往有长度限制，不能发送非常多请求，二是，谷歌搜索这一类在url中传递搜索条件，利于搜索者们进行方便分享。用post传输只是比get传输泄漏信息较难些罢了，如果在 get 中传输密码，很容易被他人查看，或用户不慎分享了 url 就会造成损失。然而，如果没有部署 ssl 服务，两者都是不安全的

对于 Apache，PHP有一个直接模块接口。如果 PHP 不能作为模块支持 web 服务器，可以将其作为 FastCGI 处理器来使用，这就意味着 CGI 可执行程序来处理服务器上的所有 PHP 文件请求

纯PHP脚本不加 ?> 结尾标记的原因是，PHP输出时会自动删除结束符后的第一个换行，如果结束符后有5个换行，则会输出4个空行

结束标记在单行注释中会被解析（/ #），在多行注释（/**/）中不会

类型

四种标量：

  • boolean
  • integer
  • float
  • string

三种复合类型：

  • array
  • object
  • callback

两种特殊类型：

  • resource
  • NULL

如果想查看某个表达式的值和类型，用 var_dump() 查看，想得到类型用 gettype()，想检验是否是某个类型用 is_type 函数

Is_int for integers，is_bool for boolean，is_boolean won’t work

Boolean 布尔类型

TRUE 或 FALSE，两个都不区分大小写，建议大写

转换为 boolean 时，以下值会被认为是 false:

  • 布尔值 false
  • 整型值 0
  • 浮点型值 0.0
  • 空字符串，以及字符串 “0” // take care
  • 不包括任何元素的数组
  • 特殊类型 NULL

这里注意：

有的时候 true 显示成 1， false 显示成 0，不能这么表达“布尔代表 0 或者 1”

Integer 整型

  • 十进制
  • 八进制，数字前面必须加上0（零）
  • 十六进制，数字前面必须加上 0x
  • 二进制，数字前面必须加上 0b

如果给定的一个数或运算结果超出了 integer 范围，将会被解释为 float

PHP 中没有整除运算符

转换为整型：

  • 布尔值。false 转为 0， true 转为 1

  • 浮点型。转换成整数时，向下取整

Float 浮点型

十进制表示一些有理数，如 0.1 或 0.7 时，无论有多少尾数都无法使用二进制精确表示，因此不能在不丢失一点精度的情况下转换为二进制。例如，floor((0.1+0.7)*10) 通常会返回 7 而不是预期中的 8，因为该结果内部的表示其实是类似 7.9999999999999991118…。

永远不要相信浮点数精确到了最后一位，也永远不要比较两个浮点数是否相等

String 字符串

表达字符串的 4 种表达方式：

  • 单引号。只可以转义单引号本身和反斜线，在单引号中内容换行，输出时也是换行的。不会解析 \n \r 之类
  • 双引号。最重要的特征是可以解析变量，还可以转义 \n \r \t
  • heredoc 语法结构
  • nowdoc 语法结构
